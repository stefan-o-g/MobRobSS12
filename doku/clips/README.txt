001-linie.mp4:
    Roboter erhält per WLAN die Aufforderung, das Programm
    "Linie" zu starten. Dieses Programm lässt den Bot einer Linie folgen.

002-motte.mp4:
    Roboter erhält per WLAN die Aufforderung, das Programm
    "Motte" zu starten. Dieses Programm lässt den Bot einer Lichtquelle
    folgen.

003-acht.mp4:
    Roboter erhält per WLAN die Aufforderung, das Programm
    "Acht" (8) zu starten. Dieses Programm lässt den Bot
    eine 8-Form fahren. Mit einer IR-Fernbedinung ist die 8-Form
    anpassbar.
    
004-programmuebertragung-motor-demo-mp4:
    Roboter erhält per WLAN ein Tokenfile (siehe Dokumentation).
    Dieses wird von einem Compiler(aus der Vorlesung "Compilerbau"
    entstanden) auf dem Bot verarbeitet und interpretiert.
    Das Programm enthält einen Befehl um einen Motor drehen zu lassen.

005-programmuebertragung-sensor-demo-mp4:
    Roboter erhält per WLAN ein Tokenfile (siehe Dokumentation).
    Dieses wird von einem Compiler(aus der Vorlesung "Compilerbau"
    entstanden) auf dem Bot verarbeitet und interpretiert.
    Das Programm gibt den Wert des linken Lichtsensors aus.

006-freie-fahrt(einzel-und-broadcast):
    Zunächst wir die IP eines einzelnen Bots auf dem Computer gesetzt.
    Danach kann man diesen vom PC aus mit W-A-S-D Fernsteuern.
    Als zweites wird die IP auf die Broadcast IP 192.168.0.255
    geändert. Jetzt empfangen alle Roboter die Steueranweisungen und
    reagieren dementsprechend synchron.
